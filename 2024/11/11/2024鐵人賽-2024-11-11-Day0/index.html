『Day 0』踏上 React 學習之旅：一段重新認識自我的過程

Posted on 2024-11-11 Edited on 2024-11-12 In 2024鐵人賽

在今年初規劃年度計劃時，原本就有想參加鐵人賽，並計畫以 React 技術系列作為寫作主題。然而，最終這系列文章並沒有在鐵人賽上發表。

為什麼要寫這個系列

在今年初規劃年度計劃時，原本就有想參加鐵人賽，並計畫以 React 技術系列作為寫作主題。然而，最終這系列文章並沒有在鐵人賽上發表。除了準備內容不夠充分和啟動時間太晚之外，更大的原因是在新工作環境中迷失了一段時間。

轉職後的這段期間，漸漸發現自己開始安於現狀，失去了當初學習時的熱忱。雖然中間也曾試著調整心態，但效果始終不如預期。直到工作上準備要開始使用 React 來做開發，重新面對這個框架時，過去學習時的點點滴滴又浮現在腦海中，或許這就是一個契機，可以把之前的學習好好的統整一下，於是決定用一個比較從容的步調來完成這個寫作計劃。

與其趕著參加鐵人賽，不如好好沉澱這段時間的所思所得，希望能寫出更有深度的內容。

為何選擇官方文件作為主軸？

之所以選擇用 React 官方文件作為主要參考，就只是一個很單純的原因：這是我最初學習的起點。

記得在學習過程中，前輩們常說：「學會閱讀官方文件，比找現成解答更重要。」當時對這句話似懂非懂，但在工作中經常需要學習新知識時，才真正體會到這個建議的可貴。

回想起第一次閱讀官方文件時的困惑與不安，那種摸索的過程雖然辛苦，卻讓我慢慢建立起面對技術文件的信心。這不只是在學習一個框架，更是在培養解讀技術文件的能力。

在技術持續演進的今天，培養理解官方文件的能力，或許比記住程式碼片段更實用。在這裡，想以個人學習官方文件的經驗，提供另一種角度來認識 React，期待能為同樣在學習的夥伴帶來一點幫助。

感謝

很感謝在學習路上一起奮鬥的同學們，還有在過程中給予指導的前輩與助教。因為有你們的幫助，才能在遇到困難時繼續前進。

希望這些學習筆記，能為正在學習 React 的朋友提供一些參考。讓我們在 React 的世界中共同學習。
